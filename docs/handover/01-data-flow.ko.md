# 예약 데이터는 어디에 저장되나요?

손님이 예약을 마치면 어떤 정보가 어디에 남는지 정리한 문서입니다. 결론부터 말씀드리면, **지금은 Stripe 대시보드가 모든 예약 정보의 진실의 원천**이고, 추가로 Vercel 서버 로그에 흔적이 남습니다.

> 용어 한 줄 정리
> - **Stripe**: 결제를 대신 처리해주는 회사. 카드 정보, 영수증, 환불을 모두 Stripe가 관리합니다.
> - **웹훅(Webhook)**: 어떤 일이 벌어졌을 때 다른 서버에게 자동으로 알림을 보내주는 콜백. "결제 완료됐어요"라고 Stripe가 우리 서버를 두드려주는 방식입니다.
> - **Vercel**: 웹사이트가 실제로 돌아가고 있는 호스팅 서비스(서버 운영사).

---

## 1. 예약이 일어나는 흐름

1. 손님이 사이트에서 `/book` 페이지를 열고 이름, 이메일, 자전거 옵션 등을 입력합니다.
2. "Pay deposit" 버튼을 누르면 우리 서버(`/api/stripe/checkout`)가 **Stripe에 결제 세션을 만들어 달라고 요청**합니다.
3. 손님은 **Stripe가 직접 호스팅하는 결제 페이지**로 이동해서 카드 정보를 입력합니다. (카드 번호는 우리 서버를 절대 거치지 않습니다 — 이 부분 덕분에 PCI 같은 까다로운 인증 요건이 우리에게 오지 않습니다.)
4. 결제가 끝나면 Stripe가 **`checkout.session.completed`라는 웹훅 이벤트**를 우리 서버(`/api/stripe/webhook`)로 보냅니다. 이때 손님이 입력했던 정보(이름, 이메일, 옵션)가 메타데이터로 함께 옵니다.

---

## 2. 지금 데이터가 남는 두 곳

### (1) Stripe 대시보드 — 가장 중요합니다

[https://dashboard.stripe.com](https://dashboard.stripe.com) 에서 로그인하시면,

- **Payments 탭**에서 모든 결제 내역을 볼 수 있습니다.
- 각 결제를 클릭하면 손님 이메일, 이름, 자전거 렌탈 여부, 1인실 여부, 전화번호 등이 **메타데이터(metadata)** 항목 아래에 그대로 들어 있습니다.
- **환불**도 이 화면에서 버튼 한 번으로 처리합니다.
- 매 예약마다 Stripe가 자동으로 **알림 이메일**을 보내드리기 때문에, 결제가 들어왔는데 못 보고 지나칠 일은 없습니다.

> 즉, "이번 달에 누가 예약했지?"가 궁금하시면 Stripe 대시보드 한 군데만 보시면 됩니다.

### (2) Vercel 서버 로그 — 보조 기록

웹훅이 들어오면 우리 서버는 다음과 같은 한 줄을 로그에 남깁니다.

```
✅ Booking confirmed: { name, email, bikeRental, privateRoom, total, sessionId }
```

이 로그는 **Vercel 프로젝트 → Logs 탭**에서 보실 수 있고, **약 30일 동안만** 보관된 후 자동으로 사라집니다. 그래서 장기 기록 용도로는 적합하지 않고, "어제 결제가 잘 들어왔는지" 같은 단기 점검용입니다.

## 3. 지금 **없는 것** (의도된 한계)

코드 안에 두 줄짜리 `// TODO` 주석으로 남아 있는, 아직 만들지 않은 기능이 두 가지 있습니다. 위치는 `src/app/api/stripe/webhook/route.ts` 26번째 줄 근처입니다.

```ts
// TODO: Store booking in database (Supabase)
// TODO: Send confirmation email (Resend)
```

- **자체 데이터베이스 저장이 없습니다.** 즉, 우리 쪽 DB(예: Supabase)에 예약 표가 별도로 쌓이지 않습니다. 모든 조회는 Stripe 대시보드에서 하셔야 합니다.
- **자체 확인 이메일 발송이 없습니다.** 손님은 Stripe가 자동으로 보내는 결제 영수증 이메일만 받게 됩니다. "예약해주셔서 감사합니다" 같은 한국어/영어 맞춤 이메일은 아직 발송되지 않습니다.

## 5. 향후 옵션 (원하실 때)

지금의 흐름으로도 10명 규모 운영에는 충분하지만, 나중에 자동화를 늘리고 싶으시다면:

- **Supabase**(무료 티어로 시작 가능)에 `bookings` 테이블을 만들고, 위에서 보신 `// TODO` 두 줄을 채우면 매 예약이 우리 DB에 자동으로 쌓입니다.
- **Resend**(무료 티어로 시작 가능)로 한국어/영어 맞춤 확인 이메일을 자동 발송할 수 있습니다.

두 작업 모두 코드 몇십 줄 수준이라 개발자 한 명에게 의뢰하시면 한나절 안에 가능합니다. 바로 필요하지 않으시면 지금 흐름(Stripe 대시보드 + 알림 이메일)으로 충분히 운영하실 수 있습니다.
